# Личный финансовый ассистент

Учебный проект по дисциплине **«Введение в специальность»**.

Проект представляет собой консольное приложение на Python для учёта личных финансов. Пользователь может добавлять доходы, расходы и финансовые события, а программа сохраняет данные и показывает аналитику.

## Цель проекта

Создать программу, которая помогает пользователю вести учёт личных доходов и расходов, анализировать траты по категориям и получать рекомендации по управлению бюджетом.

## Основные возможности

- добавление доходов;
- добавление расходов;
- добавление событий и финансовых планов;
- сохранение данных в JSON-файл;
- подсчёт общего дохода, расхода и баланса;
- анализ расходов по категориям;
- предупреждение о больших тратах на развлечения;
- прогноз остатка на следующий месяц;
- получение совета от нейросети через OpenRouter.

## Используемые технологии

- Python 3;
- JSON для хранения данных;
- Git для контроля версий;
- GitHub для размещения проекта;
- OpenRouter API для обращения к нейросети;
- библиотека `openai`.

## Как запустить проект

1. Склонировать репозиторий:

```bash
git clone https://github.com/Ofrai-pro/finance-ai.git
```

2. Перейти в папку проекта:

```bash
cd finance-ai
```

3. Установить зависимость для работы с OpenRouter:

```bash
pip install openai
```

4. Запустить программу:

```bash
python main.py
```

## Настройка нейросети

Основные функции программы работают без API-ключа: можно добавлять доходы, расходы, события и смотреть аналитику.

Для работы пункта **«Нейросеть (Gemini через OpenRouter)»** нужно получить API-ключ OpenRouter и указать его в файле `src/ai_assistant.py` вместо заглушки:

```python
api_key="ТВОЙ_OPENROUTER_КЛЮЧ"
```

API-ключ не должен храниться в публичном репозитории.

## Как пользоваться программой

После запуска пользователь видит меню:

```text
1. Добавить доход
2. Добавить расход
3. Добавить событие/план
4. ИИ-помощник (аналитика и правила)
5. Нейросеть (Gemini через OpenRouter) – получить совет
6. Выход
```

Пользователь выбирает пункт меню, вводит сумму, категорию или описание. Все записи сохраняются в файл `data/records.json`.

Данные не обнуляются каждый месяц. Программа хранит все записи постоянно и анализирует сохранённые данные.

## Пример использования

Пользователь получил стипендию и добавляет доход:

```text
Сумма: 13000
Источник: стипендия
Комментарий: за май
```

Потом пользователь добавляет расход:

```text
Сумма: 750
Категория: еда
Комментарий: продукты
```

После этого можно открыть аналитику и увидеть общий доход, общий расход, баланс и распределение расходов по категориям.
